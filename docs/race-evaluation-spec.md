# 競馬予想サイト 能力評価・補正ロジック仕様

本ドキュメントは「標準評価」と「補正後評価」を分離し、馬の能力値は変更せず重みのみを調整する競馬予想 UI の要件定義である。

## 実装目的

出馬表上で各馬に対して以下を表示する。

- 標準評価
- 補正後評価
- 印
- 評価理由
- 馬の能力タイプ
- 補正による順位変動
- 同タイプで買える相手候補

**目的**: 標準能力評価に対して、馬場・展開・バイアス補正を加え、今回買うべき馬を再評価する。

## 原則

1. **馬の能力値は直接変更しない** — 変えるのは「今回のレースで求められる能力重み」。
2. **評価は2段階** — 標準評価（コース標準重み）と補正後評価（馬場・展開・バイアス後）。
3. **補正は説明可能** — 順位・印の変化に理由を付与する。

## 評価軸の位置づけとレース内相対化（設計方針）

**評価軸を捨てるのではなく、各軸で出した点を材料にし、最終的な並べ方に「同レース内相対化」を含める。**

- **スピード・スタミナ・末脚・持続力・パワー** — 馬の能力値（固定プロフィール）。
- **コース・馬場・展開・バイアス** — 今回どの能力を重く見るか（重み・補正の入力）。
- **今回向き** — 能力と今回条件の噛み合い（能力面）。
- **脚質** — 位置取り傾向。
- **展開適合** — 脚質と展開の噛み合い（位置取り面）。

旧来の「5能力×今回条件の重み＝補正後評価（絶対点）」で打ち切らず、パイプライン上で次の流れを想定する。

```text
5能力
→ 基礎能力（例: 5軸の集約スコア）
→ 今回条件スコア（重み付き合計 等）
→ レース内相対化（同じ出走表の中での比較）
→ 展開適合の加味
→ 最終評価（印・買い判断・表示用順位 等）
```

- **`raceRelativeScore`（仮名）** — 同レース内での強さ比較のための中間指標。出走メンバーが揃っていて初めて意味を持つ。

競馬は全頭が同一条件下のタイムトライアルではないため、「何点か」より「このメンバーの中でどれだけ抜けているか」が最終判断に効く。実装では、補正後スコアを**同一レース内で min-max 正規化**した `raceRelativeScore`（0〜100）に加え、**展開適合の小さなボーナス**を足した `finalEvaluationScore` を用い、**印・買い判断の順位は `finalRank`** とする（詳細は `docs/implementation-plans.md` §6）。

## 能力指標（固定）

`speed` | `stamina` | `kick` | `sustain` | `power`

表示名: スピード / スタミナ / 末脚 / 持続力 / パワー

## 標準コース重み・補正定義

数値定義は `src/domain/race-evaluation/courseWeights.ts` および `adjustments.ts` を正とする。

## 補正強度・重み制御

- 強度: `weak` 0.5 / `middle` 1.0 / `strong` 1.5（初期 `middle`）
- 各重み `MIN_WEIGHT`〜`MAX_WEIGHT` にクランプ後、合計 1.0 に正規化

## 受け入れ条件

- 標準評価と補正後評価が別々に算出される
- 馬場・展開・バイアスをユーザーが変更できる
- 補正後にランキングが再計算される
- 補正前後の差分・評価理由が表示される
- 能力値自体は補正で変更されない
- 重みは正規化後に合計 1.0、各重みは所定レンジ内

## スコープ外

勝率・期待値、オッズ連携、レース狙い度の数値実装（狙い度は将来用の定義のみ）。

詳細な文言・ UI 要件はプロジェクト起票時の実装プロンプトに準拠する。
