# Nintendo Pop Design Spec (競馬AIサイト)

## 1. 目的

本仕様は、現行の Apple 風トーンを、Nintendo サイトのような「にぎやか・ポップ」な体験へ段階的に移行するための実装基準を定義する。  
対象は `RaceDetailView` を中心とした詳細ページUIと、その共通デザイントークンである。

## 2. デザイン方針

- テイスト: カジュアル / にぎやか・ポップ
- ブランド基調: 赤を主役、白をベース
- UX方針: 情報量は維持しつつ、視認性と回遊性を強化
- 実装方針: 既存のDOM構造は活かし、まずは CSS 主体で刷新

## 3. カラートークン

### Core

- Brand Red: `#e60012`
- Brand Red Dark: `#c60010`
- Brand Red Light: `#ff3348`
- Base White: `#ffffff`
- Soft White: `#fff6f7`

### Text / Border

- Primary Text: `#1f1f24`
- Secondary Text: `#565a66`
- Muted Text: `#7b8190`
- Border: `#f1c6cc`
- Border Strong: `#e7909a`

### Usage Rule

- 重要CTA、アクティブ状態、フォーカス、主要タブに赤を使用
- 背景は白を基調に、セクション区切りで薄い赤を使用
- 補助情報は濃いグレーで可読性を優先

## 4. コンポーネント仕様（第1段階）

### 4.1 ヘッダー (`.app__hero`)

- 白ベース + 薄い赤グラデーション
- 見出しは太字化（700以上）
- パンくずやメタ情報の一部に赤アクセント

### 4.2 条件アコーディオン (`.condition-accordion`)

- ボーダー強化（1px -> 2px 相当の視認性）
- 折りたたみトリガーは白背景 + 赤のフォーカスリング
- hover で薄赤背景に変化

### 4.3 タブ (`.view-tabs`)

- ラインタブから「ボタンタブ」に変更
- アクティブタブは赤背景 + 白文字 + 軽い影
- 非アクティブタブは白背景 + 枠線で段差を作る

### 4.4 一覧・カード

- 行 hover やカード hover を薄赤トーン化
- カードの角丸と境界をやや強調してポップ感を追加
- 結果を示す強調要素は赤軸で統一

## 5. レイアウト / モーション

- 既存の2カラム構成は維持
- sticky UI は維持し、背景のみブランドトーンに更新
- hover / active のトランジションは 120ms - 220ms の範囲で統一

## 6. 適用対象ファイル（第1段階）

- `src/index.css`
- `src/components/race/RaceDetailView.tsx`（必要時のみ最小変更）

## 7. 受け入れ基準

- 主要導線（ヘッダー、条件、タブ、一覧/カード）で赤×白の統一感がある
- テキストコントラストを損なわず、情報可読性を維持
- 既存機能（タブ切替、条件更新、結果確認）に影響がない
- モバイル/デスクトップ双方でUI崩れがない

## 8. 実装メモ

- 既存変数 `--c-blue*` は互換性維持のため残し、値を赤系に再定義する
- 段階導入として、まず詳細ページのビジュアル優先要素から更新する
- 今後、必要に応じて一覧ページ (`rl-*`) まで同じトーンへ横展開する

## 9. 追加ガイド（UMAJO x Nintendo x Dashboard）

### 9.1 配色の拡張ルール

- 基本は同系色（白 / ピンク系 / ミント系）で構成し、色数を抑える
- 重要指標（◎、期待値上位、注意喚起）だけ補色アクセントで強調する
- 意味のある色（枠色・買い判定）を優先し、装飾色の乱用は避ける

### 9.2 タイポ / 装飾

- 本文フォントは可読性重視で 1 系統
- 見出しや注目タグは別テイスト（手書き風に見える演出）を限定利用
- 芝・馬・蹄鉄など競馬モチーフは、アイコンとして軽く添える

### 9.3 出馬表（表UI）

- 枠線依存を減らし、余白を増やして視線誘導を改善する
- 行の背景はストライプ（近い明度差）で判別しやすくする
- テキスト情報は可能な限りバッジ化して視認を高速化する

### 9.4 モーション

- ナビゲーションに軽い遊び心（小さな走るモチーフ）を加える
- スコアバーやグラフは初回表示時に伸長アニメーションを付与する
- 演出は 120ms〜600ms 以内で、情報読解を妨げない範囲に制御する

## 10. カード / ボタン最適化ルール

### 10.1 カード情報設計

- カード内は要素を詰めすぎず、余白比率を増やして圧迫感を抑える
- 最重要情報（印、軸、本命）を左上に寄せ、視線導線を安定させる
- テキスト主体からバッジ主体へ移行し、判読コストを下げる
- カード枠は角丸大きめ + やわらかい点線で手作り感を演出する

### 10.2 表示密度切替

- 詳細カード群の右上に「通常 / コンパクト」切替ボタンを設置
- コンパクト時は補助説明を省略し、主要スコアと判定を優先表示する
- モバイルでも押しやすい高さ（42px 以上）を基準とする

### 10.3 ボタンUX

- 主要ボタンは 44px 前後の高さを確保してタップしやすくする
- hover / active で押下感（明度変化・微小移動）を必ず付与する
- 強調ボタンはグラデーションや影で「行動導線」を明示する

### 10.4 買い目カードの可視化

- 馬券種ごとにカード化し、点数・合計・想定払戻を同一カードに集約
- 配分比率はドーナツ表示などで視覚化し、資金配分の理解を高速化する
