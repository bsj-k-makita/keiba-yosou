```text

↓
```

# 新規実装：変換レイヤー

## 入力

```ts
analysisJson
```

（`src/lib/race-data/analysisJsonTypes.ts` に構造定義。パイプライン由来の生オブジェクトを `unknown` から畳み込む）

## 出力

```ts
type RaceEvaluationData = {
  raceId: string;
  raceInfo: RaceInfo;
  condition: RaceCondition;
  entries: RaceEntryEvaluation[];
};
```

## RaceEntryEvaluation

```ts
type RaceEntryEvaluation = {
  horseId: string;
  horseName: string;
  horseNumber: number;

  runningStyle: RunningStyle;

  abilities: {
    speed: number;
    stamina: number;
    kick: number;
    sustain: number;
    power: number;
  };

  abilityGrades: {
    speed: AbilityGrade;
    stamina: AbilityGrade;
    kick: AbilityGrade;
    sustain: AbilityGrade;
    power: AbilityGrade;
  };

  evaluation: {
    baseScore: number;
    adjustedScore: number;
    scoreDiff: number;
    fitLevel: "高" | "中" | "低";
    paceFit: "◎" | "○" | "△" | "×";
    buyLabel: BuyLabel;
  };
};
```

（実装型は `raceEvaluationTypes.ts` を参照。等級は既存 `DisplayGrade` 系に正規化）

---

# 変換処理の内容

1. 出馬表情報の抽出（馬名・馬番・騎手・斤量・枠）
2. 脚質の推定（通過順・先行率・過去走位置取り。パイプライン済みを優先）
3. 能力値算出（スピード / スタミナ / 末脚 / 持続力 / パワー）
4. 能力ランク化（S / A+ / A / B / C 等。不足時は同出走内で相対化 `computeAbilityLetterGrades`）
5. 評価計算（base / adjusted / diff。不足時は `buildEvaluationData` へ）
6. 今回向き（能力 × 条件重み。不足時に算出）
7. 展開適合（脚質 × ペース × バイアス。不足時に算出）
8. 買い判断（パイプラインで埋まっていなければドメインの `evaluateRace` 経由で付与）

---

# 実装順序

```text
1. RaceEvaluationData 型定義
2. convertToRaceEvaluationData 作成
3. 旧JSON読み込み処理を実装
4. モックデータを廃止
5. JSON → UI 表示確認
6. 能力算出ロジック調整
7. 脚質推定精度改善
8. netkeiba取得処理を再接続
```

（※ コード実装箇所：`src/lib/race-data/convertToRaceEvaluationData.ts`、`raceDataRepository.ts`）
