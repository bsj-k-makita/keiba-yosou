# 馬券的中ロジックまとめ

このドキュメントは、現行実装で「どの買い目を買って」「どう金額配分し」「どう的中判定・払戻計算するか」を整理したものです。

## 1. 集計対象（何を当たりとして数えるか）

- 集計対象は **EV推奨券（`evTickets`）のみ**。
- 印（◎○▲☆△）は表示・診断に使うが、**印フォーメ自体は集計対象にしない**。
- そのため、印が無い馬でも `evTickets` に入っていれば的中対象。

## 2. 軸（◎）の安全フィルター

- 軸として使う ◎ は、予測勝率が **8%以上**（`ANCHOR_MIN_PREDICTED_WIN_RATE = 0.08`）の馬のみ。
- 勝率 8% 未満の馬は、高オッズでも軸から除外（相手側には残りうる）。

## 3. EV推奨券の作り方（券種別）

EV推奨券は `generateTicketsFromEvaluation()` で生成する。

### 3.1 TSモード

- 単勝（WIN）: `prob × odds` を使用
- 馬連（REN）: `estimatePairProbability` を使用
- ワイド（WREN）: `estimatePairProbability` + 相手単勝オッズ下限
- 3連複（TRI）: `estimateTrifectaProbability` を使用
- 全券種で **動的EV閾値** を適用:
  - `必要EV = ベース閾値 + (0.05 / 予測勝率)`
  - 低勝率ほど厳しくなる

### 3.2 AIモード

- ◎軸固定で、相手候補を `ai_effective_ev` 順で選定し、券を生成
- 2列目の脚質分散ロジックあり（環境変数 `BETTING_STYLE_DIVERSIFY_PATTERN` でA/B/C切替）
- 3連複は候補生成後にEV順で上位点へ圧縮

## 4. 金額配分（ケリー基準の傾斜買い）

`evTicketsToBetTickets()` で、券ごとに金額（`betAmount`）を決定する。

- ケリー式（フル）:
  - `f = (p*b - q) / b`
  - `p`: 予測勝率、`b`: オッズ-1、`q`: 1-p
- 実運用係数（券種別）:
  - WIN / REN / WREN: **0.25**
  - TRI: **0.15**
- レース基準予算:
  - `KELLY_BASE_BUDGET_PER_RACE = 10,000`
- 金額制約:
  - 最低 `100` 円
  - 上限 `3,000` 円
  - `100` 円単位で丸め

## 5. 1レース総投資キャップ

- 1レースの合計投資は `MAX_INVESTMENT_PER_RACE`（現設定 `8,000`）で上限管理
- 上限超過時は、以下で圧縮:
  1. 各券の最低額 `100` 円を確保
  2. 余剰部分を比例縮小
  3. 100円単位で再丸め
  4. まだ超過する場合は高額券から100円ずつ減額

## 6. 的中判定の基本

`calculateRacePayout()` が判定・払戻計算を担当。

- WIN: 1着と買い目1頭が一致
- MAIN_LINE（馬連）: 1-2着の2頭集合に買い目2頭が含まれる（順不同）
- WIDE（ワイド）: 1-3着の3頭集合に買い目2頭が含まれる（順不同）
- TRIFECTA_FORM（3連複）: 1-3着の3頭集合に買い目3頭が一致（順不同）

## 7. 公式払戻データの優先

- 公式払戻プールがある券種は、**配当行一致のみ**を的中として扱う
- 着順だけ合っても、配当行に無い組み合わせは不的中
- 公式払戻が無い券種のみ、着順判定 + フォールバックオッズで払戻推定

## 8. 着順データの作り方

- 基本: `horseId -> horseNumber` で解決
- フォールバック: `place.horseNumber` を利用
- ID不整合時も馬番が取れれば計算継続

## 9. 「◯◎☆で決着」の扱い

結論: **印だけでは的中にならない**。`evTickets` にその組み合わせが入っているかで決まる。

- WIN: ◎が2着なので不的中
- 馬連（MAIN_LINE）: `◎-◯` がEV推奨で買われていれば的中
- ワイド（WIDE）: `◎-◯` または `◎-☆` が買われていれば的中
- 3連複（TRIFECTA_FORM）: `◎-◯-☆` が買われていれば的中

つまり、同じ「◯◎☆決着」でも、レースごとの `evTickets` 構成によって的中券種は変わる。
