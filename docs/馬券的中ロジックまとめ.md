# 馬券的中ロジックまとめ

このドキュメントは、現行実装で「どの買い目を買って」「どう金額配分し」「どう的中判定・払戻計算するか」を整理したものです。

## 1. 集計対象（何を当たりとして数えるか）

- 集計対象は **EV推奨券（`evTickets`）のみ**。
- 印（◎○▲☆△）は表示・診断に使うが、**印フォーメ自体は集計対象にしない**。
- そのため、印が無い馬でも `evTickets` に入っていれば的中対象。

## 2. ◎（topMarkGate）の位置づけ

- `resolveTopMarkGate` / `ANCHOR_MIN_PREDICTED_WIN_RATE`（8%）は **UI・診断用**（◎表示の妥当性チェック等）。
- **EV推奨券の生成条件には使わない**（2026-05-23 以降）。複勝券も ◎ を含む必要はない。

## 3. EV推奨券の作り方（券種別）

EV推奨券は `generateBetTicketsFromEvaluation()` → `generateTicketsFromEvaluation()` で生成する。  
**TS / AI 共通**の純粋EVロジック（フォーメーション分岐なし）。

### 3.1 対象馬・組み合わせ

- 勝率 ≥ 1%（`VALID_PROB_THRESHOLD = 0.01`）の馬のみ対象
- **単勝（WIN）: ◎（`topMarkGate`）のみ**（EV ≥ 1.3 かつ予測勝率8%以上で軸成立時）。最大1点
- 馬連・ワイド: 有効馬の **全2頭組み合わせ**
- 3連複: 有効馬の **全3頭組み合わせ**

### 3.2 合成確率（Harville）

- 馬連・ワイド（ペア）: `estimatePairProbability`
  - `P(A,B) = P(A)×P(B)/(1-P(A)) + P(B)×P(A)/(1-P(B))`
- 3連複: `estimateTrifectaProbability`
  - `P(A→B→C) = P(A) × P(B)/(1-P(A)) × P(C)/(1-P(A)-P(B))` を **[A,B,C] の全6順列** で合算

### 3.3 EV と閾値

- `EV = 合成確率 × オッズ`
- オッズ: 実オッズ優先。欠損時は `buildOddsMapForEvEvaluation` の推定オッズ（Harville + 市場乖離係数）
- 固定閾値（`investmentEvConstants.ts`）:

| 券種 | 閾値 |
|------|------|
| WIN | 1.3（`DEFAULT_EV_THRESHOLD`） |
| REN | 1.60 |
| WREN | 1.20 |
| TRI | 1.50 |

### 3.4 点数キャップ

EV降順で券種別上限（`capEvTicketsByType`）:

| 券種 | 上限 |
|------|------|
| WIN | 10点 |
| REN | 10点 |
| WREN | 5点 |
| TRI | 3点 |

### 3.5 撤廃済みロジック（参考）

2026-05-23 以前に存在した以下は **EV券生成から削除**:

- ◎軸必須（複勝券）
- AIモード `createFixedAiFormationTickets`（EV順2列目・脚質分散・印付き相手優先）
- ワイド相手の単勝オッズ下限
- 動的EV閾値（`base + 0.05 / p`）
- 3連複0点時の全滅回避1点

## 4. 金額配分（ケリー基準の傾斜買い）

`evTicketsToBetTickets()` で、券ごとに金額（`betAmount`）を決定する。

- ケリー式（フル）:
  - `f = (p*b - q) / b`
  - `p`: 予測勝率、`b`: オッズ-1、`q`: 1-p
- 実運用係数（券種別）:
  - WIN / REN / WREN: **0.25**
  - TRI: **0.15**
- レース基準予算:
  - `KELLY_BASE_BUDGET_PER_RACE = 10,000`
- 金額制約:
  - 最低 `100` 円
  - 上限 `3,000` 円
  - `100` 円単位で丸め

## 5. 1レース総投資キャップ

- 1レースの合計投資は `MAX_INVESTMENT_PER_RACE`（既定 `8,000`）で上限管理
- ストレステスト時のみ、環境変数 `BETTING_MAX_INVESTMENT_PER_RACE` で一時上書き可能（例: `5000`）
- 上限超過時は、以下で圧縮:
  1. 各券の最低額 `100` 円を確保
  2. 余剰部分を比例縮小
  3. 100円単位で再丸め
  4. まだ超過する場合は高額券から100円ずつ減額

## 6. 的中判定の基本

`calculateRacePayout()` が判定・払戻計算を担当。

- WIN: 1着と買い目1頭が一致
- MAIN_LINE（馬連）: 1-2着の2頭集合に買い目2頭が含まれる（順不同）
- WIDE（ワイド）: 1-3着の3頭集合に買い目2頭が含まれる（順不同）
- TRIFECTA_FORM（3連複）: 1-3着の3頭集合に買い目3頭が一致（順不同）

## 7. 公式払戻データの優先

- 公式払戻プールがある券種は、**配当行一致のみ**を的中として扱う
- 着順だけ合っても、配当行に無い組み合わせは不的中
- 公式払戻が無い場合のみ、着順判定で的中可否を見て払戻を推定する
  - WIN: `winOddsByNumber` から推定
  - 馬連/ワイド/3連複: `fallbackExoticOdds` から推定

## 8. 着順データの作り方

- 基本: `horseId -> horseNumber` で解決
- フォールバック: `place.horseNumber` を利用
- ID不整合時も馬番が取れれば計算継続

## 9. 「◯◎☆で決着」の扱い

結論: **印だけでは的中にならない**。`evTickets` にその組み合わせが入っているかで決まる。

- WIN: ◎が2着なので不的中（◎単勝を買っていれば）
- 馬連（MAIN_LINE）: `◎-◯` がEV推奨で買われていれば的中（**◎軸でなくても** `◯-☆` 等が推奨されていれば的中）
- ワイド（WIDE）: 推奨に含まれる2頭が1-3着に入れば的中
- 3連複（TRIFECTA_FORM）: 推奨に含まれる3頭が1-3着に一致すれば的中

つまり、同じ「◯◎☆決着」でも、レースごとの `evTickets` 構成（EV上位組み合わせ）によって的中券種は変わる。
