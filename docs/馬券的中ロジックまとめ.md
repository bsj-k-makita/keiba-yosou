# 馬券的中ロジックまとめ

このドキュメントは、現行実装で「どの買い目を買って」「どう的中判定し」「どう払戻を計算するか」を整理したものです。

## 1. 何を集計対象にしているか

- 集計対象は **EV推奨券（`evTickets`）のみ**。
- 印フォーメーション（◎○▲☆△）は、診断や表示には使うが、払戻集計の主判定には使わない。
- つまり、印が付いていない馬番でも、EV推奨券に含まれていれば的中対象になる。

## 2. EV推奨券の生成ルール（概要）

EV推奨券は `generateTicketsFromEvaluation()` で生成する。

- 単勝（WIN）: `prob × odds`（AIモードは `ai_effective_ev`）で閾値以上
- 馬連（REN / UI上は MAIN_LINE）: ◎軸の2頭組み合わせでEV閾値以上
- ワイド（WREN / UI上は WIDE）: ◎軸の2頭組み合わせでEV閾値以上
  - 追加条件: 相手馬の単勝オッズが `10.0` 以上（大穴限定トリミング）
- 3連複（TRI / UI上は TRIFECTA_FORM）: ◎軸3頭組み合わせでEV閾値以上
  - EV閾値は `1.85`
  - 1レースあたり最大 `5` 点

## 3. 的中判定の基本

的中計算は `calculateRacePayout()` が担当する。

券種別の「着順ベース判定」は次の通り:

- WIN: 1着と買い目1頭が一致
- MAIN_LINE（馬連）: 1-2着の2頭集合に買い目2頭が含まれる（順不同）
- WIDE（ワイド）: 1-3着の3頭集合に買い目2頭が含まれる（順不同）
- TRIFECTA_FORM（3連複）: 1-3着の3頭集合に買い目3頭が一致（順不同）

## 4. 公式払戻データがある場合の優先ルール（重要）

現行仕様では、**公式払戻データがある券種は、配当行一致のみを的中扱い**にする。

- 例: 馬連プール（`REN`）が存在するレースでは、買い目が `REN` 配当行に存在する場合だけ `hitCount` を加算。
- 着順だけ合って見えても、配当行に無い組み合わせは的中扱いにしない。

意図:

- 「着順上は当たりっぽいが、公式配当上は不的中」の誤判定を防ぐため。

## 5. 公式払戻が無い場合のフォールバック

公式払戻データが未取得の券種のみ、次を使う:

- 着順ベースで的中判定
- 払戻はフォールバック推定オッズ（`fallbackExoticOdds`）から計算

## 6. 着順（finishOrder）の構築

`finishOrder` は結果データから馬番配列を作る。

- 基本: `horseId -> horseNumber` 解決
- フォールバック: `horseId` が解決できない場合は `place.horseNumber` を使用

これにより、結果データのID不整合があっても、馬番があれば的中計算を継続できる。

## 7. 表示ラベルについて

誤解を避けるため、結果画面とレポートの表記は次に統一している:

- `馬連EV（◎軸）`
- `ワイドEV（◎軸）`
- `3連複EV（◎軸）`

これは「印フォーメの買い目」ではなく、「EV推奨券の実買い目」を集計していることを明示するため。

## 8. 具体例（202608030806）

- 実着順: `8-6-4`
- 公式馬連配当: `6-8`
- EV推奨券に `6-8` が含まれていたため、馬連は的中扱いになる
- `6` や `4` に印が無くても、EV推奨券に含まれていれば的中しうる
